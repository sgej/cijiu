# 云端 iOS 打包方案对比

## 方案总览

| 方案 | 免费额度 | 难度 | 推荐指数 | 说明 |
|------|---------|------|---------|------|
| GitHub Actions | 每月2000分钟 | ⭐⭐ | ⭐⭐⭐⭐⭐ | 最推荐，与Git集成好 |
| Codemagic | 每月500分钟 | ⭐ | ⭐⭐⭐⭐ | 专为移动应用设计，简单易用 |

## 方案一：GitHub Actions（推荐）

### 优点 ✅
- 免费额度充足（每月2000分钟）
- 与GitHub仓库深度集成
- 配置灵活，可自定义
- 社区资源丰富
- 项目已内置配置文件

### 缺点 ❌
- 需要一定的YAML配置知识
- macOS runner 启动较慢（约2-3分钟）

### 免费额度
- **公开仓库**: 无限分钟数
- **私有仓库**: 每月2000分钟
- macOS 构建消耗分钟数: 10倍系数（1分钟实际=10分钟额度）

### 配置方法
项目已包含 `.github/workflows/build.yml` 配置文件，只需：
1. 将代码推送到GitHub仓库
2. 触发工作流（push或手动）
3. 下载构建产物

详细步骤见 `README_PACKAGE.md` 中的「无Mac云端打包指南」。

## 方案二：Codemagic

### 优点 ✅
- 专为移动应用设计，操作简单
- 可视化配置界面
- 支持自动签名
- 内置测试和分发功能
- 支持多种平台（iOS/Android/Flutter等）

### 缺点 ❌
- 免费额度较少（每月500分钟）
- macOS 构建消耗较快
- 高级功能需要付费

### 免费额度
- 每月 500 分钟
- 无需信用卡

### 配置步骤

#### 1. 注册账号
1. 访问 [codemagic.io](https://codemagic.io/)
2. 使用 GitHub/GitLab/Bitbucket 账号登录
3. 授权访问你的仓库

#### 2. 添加应用
1. 点击 "Add application"
2. 选择你的代码仓库
3. 选择项目类型（iOS）
4. 点击 "Finish"

#### 3. 配置构建
1. 在应用设置中选择 "Workflow Editor"
2. 配置构建步骤：
   - **Build**: 选择 "Xcode build"
   - **Scheme**: 选择 MineRadio
   - **Configuration**: 选择 Release
   - **Build format**: 选择 IPA

#### 4. 代码签名（可选）
1. 在 "Distribution" 部分配置
2. 上传你的证书和描述文件
3. 或使用 Codemagic 的自动签名功能

#### 5. 开始构建
1. 点击 "Start new build"
2. 等待构建完成
3. 下载 IPA 文件

- 1 个项目
- 无需信用卡

### 配置步骤

#### 1. 注册账号
1. 访问 [appcircle.io](https://appcircle.io/)

1. 选择 "Build" 模块
2. 连接你的 Git 仓库
3. 选择分支

#### 4. 配置构建
1. 选择 Xcode 版本
2. 配置 Scheme 和 Configuration
3. 配置签名（可选）

#### 5. 运行构建
1. 点击 "Start Build"
2. 等待构建完成
3. 下载构建产物

- 官网: [macstadium.com](https://www.macstadium.com/)
- 价格: 约 $79/月起
- 优点: 完全控制，性能好
- 缺点: 价格较高

- 价格: 约 $1.083/小时
- 优点: 按需付费，弹性扩展
- 缺点: 价格昂贵，最小租用24小时

## 未签名 IPA 安装方案

如果没有开发者账号，生成的未签名 IPA 可以通过以下方式安装：

### 1. TrollStore（推荐）
- **支持系统**: iOS 14.0 - 16.6.1
- **优点**: 永久签名，无需电脑，无需越狱
- **缺点**: 有系统版本限制
- **详细说明**: 见 `README_PACKAGE.md`

### 2. 免费 Apple ID 签名
- **支持系统**: 所有 iOS 版本
- **优点**: 官方方式，安全可靠
- **缺点**: 7天过期，需要重新签名
- **工具**: AltStore、Sideloadly

### 3. 企业签名
- **支持系统**: 所有 iOS 版本
- **优点**: 有效期长（1年），无需设备UDID
- **缺点**: 需要企业开发者账号（$299/年）
- **注意**: 共享企业证书可能被苹果封禁

### 4. 越狱设备
- **支持系统**: 可越狱的 iOS 版本
- **优点**: 完全自由，无需签名
- **缺点**: 有安全风险，可能失去保修
- **工具**: Cydia、Sileo 等包管理器

### 5. TestFlight（需要开发者账号）
- **支持系统**: 所有 iOS 版本
- **优点**: 官方测试平台，安全可靠
- **缺点**: 需要开发者账号，90天过期

## 常见问题

### Q: 免费额度用完了怎么办？
A: 可以注册多个平台的账号，或者使用不同的 GitHub 账号。

### Q: 构建失败怎么办？
A: 
1. 查看构建日志，定位错误原因
2. 检查项目配置是否正确
3. 确保 Xcode 版本兼容
4. 参考各平台的文档和社区

### Q: 未签名 IPA 能直接安装吗？
A: 不能直接安装，需要通过签名工具或特殊方式安装。详见「未签名 IPA 安装方案」。

### Q: 哪个方案最稳定？
A: GitHub Actions 和 Codemagic 都比较稳定，推荐优先尝试。

### Q: 可以同时使用多个方案吗？
A: 可以，建议同时配置 GitHub Actions 和 Codemagic，互为备份。
