# MineRadio iOS 版本 - 技术方案说明

## 一、APK 分析结果

### 1.1 应用基本信息
- **应用名称**: MineRadio
- **版本号**: v1.1.4
- **包大小**: 约 34MB
- **支持架构**: arm64-v8a (64位ARM)

### 1.2 核心功能
经过反编译分析，MineRadio 是一款功能丰富的 3D 音乐可视化播放器，主要功能包括：

1. **3D 音乐可视化**
   - 基于 Three.js 的 3D 渲染引擎
   - 骷髅头等 3D 模型随音乐律动
   - 多种视觉效果：泛光、粒子、扭曲、散射等
   - 音乐节拍实时检测（music-tempo 库）

2. **多平台音乐支持**
   - QQ 音乐
   - 网易云音乐
   - 酷狗音乐
   - 汽水音乐
   - 本地音乐播放

3. **AI 手势交互**
   - 基于 MediaPipe + TensorFlow Lite
   - 手部关键点实时检测
   - 多种手势识别（张开、握拳、指向等）
   - 手势控制播放

4. **桌面歌词**
   - 悬浮歌词显示
   - 歌词特效（发光、粒子效果）
   - 自定义字体、颜色、大小

5. **动态壁纸**
   - 音乐可视化壁纸
   - 自定义背景图片/视频

6. **丰富的自定义选项**
   - 视觉效果参数调节
   - UI 主题自定义
   - 玻璃拟态界面设计

### 1.3 技术架构
Android 版本采用混合开发架构：
- **原生层**: Java/Kotlin 提供系统功能
- **Web 层**: HTML5 + JavaScript 实现 UI 和 3D 可视化
- **AI 层**: MediaPipe + TFLite 实现手势识别
- **桥接层**: JavaScriptInterface 实现 JS 与原生通信

## 二、iOS 转换技术方案

### 2.1 架构选型
采用与 Android 版本类似的混合架构，最大程度复用 Web 层代码：

```
┌─────────────────────────────────────────┐
│              Web 层 (复用)              │
│  HTML5 + Three.js + 音乐可视化逻辑      │
└─────────────────┬───────────────────────┘
                  │
         ┌────────▼────────┐
         │   JS 桥接层     │
         │  (iOS 适配)     │
         └────────┬────────┘
                  │
┌─────────────────▼───────────────────────┐
│           iOS 原生层 (重写)              │
│  Swift + WKWebView + AVFoundation       │
│  + Vision 手势识别 + 系统 API            │
└─────────────────────────────────────────┘
```

### 2.2 核心技术栈

| 模块 | Android 技术 | iOS 技术 | 说明 |
|------|-------------|----------|------|
| Web 容器 | WebView | WKWebView | 性能更好，功能更强 |
| JS 桥接 | JavascriptInterface | WKScriptMessageHandler | 更安全的通信方式 |
| 音乐播放 | MediaPlayer | AVPlayer | 原生播放器 |
| 手势识别 | MediaPipe + TFLite | Vision 框架 | iOS 原生 AI 框架 |
| 后台播放 | Service | AVAudioSession | 后台音频模式 |
| 锁屏控制 | MediaSession | MPRemoteCommandCenter | 系统级远程控制 |

### 2.3 JS 桥接适配方案

由于 Android 和 iOS 的 JS 桥接方式不同，我们创建了 `ios-bridge.js` 适配层：

1. **兼容 Android 接口**: 创建 `window.Android` 对象，保持接口一致
2. **消息队列**: 原生环境就绪前缓存消息
3. **事件分发**: 统一的原生事件处理机制
4. **功能检测**: 提供 `hasFeature()` 方法检测平台能力

### 3.2 部分实现功能 ⚠️

- **第三方音乐平台**: 需要单独适配各平台 API，未包含在本项目中
- **桌面歌词**: iOS 系统限制，无法实现悬浮窗歌词
- **动态壁纸**: 需要通过 Live Photo 或其他方式实现，未包含在本项目中

### 3.3 技术限制 ❌

由于 iOS 系统限制，以下功能无法实现或受限：

1. **桌面悬浮歌词**
   - iOS 不允许应用在其他应用上方显示悬浮窗
   - 替代方案：通知中心歌词、锁屏歌词

2. **动态壁纸**
   - iOS 动态壁纸有严格的系统限制
   - 替代方案：Live Photo、静态壁纸

3. **后台运行限制**
   - 仅音频可后台播放
   - 3D 渲染在后台会暂停

4. **文件系统访问**
   - iOS 应用有沙盒限制
   - 只能访问用户授权的文件

5. **第三方音乐 API**
   - 各音乐平台 API 可能有平台差异
   - 需要单独适配登录和播放接口

## 五、编译与运行说明

### 5.1 环境要求
- Xcode 14.0+
- iOS 14.0+
- Swift 5.0+
- Apple Developer 账号（可选，用于真机调试）

### 5.2 编译步骤

1. **打开项目**
   ```bash
   open MineRadio.xcodeproj
   ```

2. **配置签名**
   - 在 Xcode 中选择项目
   - 进入 "Signing & Capabilities"
   - 选择你的 Team
   - 修改 Bundle Identifier 为唯一值

3. **选择目标设备**
   - 选择模拟器或真机
   - 建议使用真机测试手势识别功能

4. **编译运行**
   - 点击 Run 按钮或按 Cmd+R

### 5.3 功能测试

1. 点击导入本地音乐
2. 选择音频文件
3. 测试播放/暂停/进度跳转

1. 在设置中开启手势识别
2. 授权相机权限
3. 在相机前做出不同手势测试

1. 播放音乐
2. 观察 3D 模型随音乐律动
3. 调节可视化参数

## 七、技术总结

本次 APK 到 iOS 的转换采用了**混合架构复用**的方案，核心思路是：

1. **最大程度复用 Web 层代码**：3D 可视化、UI 界面、业务逻辑等 Web 代码 100% 复用
2. **重写原生桥接层**：使用 iOS 原生技术重新实现系统功能
3. **平台差异适配**：通过桥接适配层抹平平台差异
4. **AI 功能原生实现**：使用 iOS Vision 框架替代 MediaPipe

这种方案的优势：
- ✅ 开发效率高，大量代码可复用
- ✅ 功能一致性好，用户体验接近
- ✅ 维护成本低，Web 层更新可跨平台同步
- ✅ 性能有保障，关键功能使用原生实现
